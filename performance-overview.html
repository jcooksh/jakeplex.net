JakePlex — Performance Overview

State of play after four optimization rounds · June 12, 2026 · numbers from code analysis, build output and multi-agent audits (≈120 agents total)
Main heavy hitter right now: nothing dominant remains in steady-state. The last big one — the Plex library walk (every search and every Library visit pulled the entire library, multi-MB over the Plex box's home uplink, 1–4 s, gating each search) — is fixed in the latest deploy with a 5-minute server cache, edge caching, and 99% smaller payloads. What's left, ranked: (1) first-visit cold path (11 render-blocking Google-Font files + serverless cold start + first library crawl), (2) background compositing (3 layers at 20 Hz while you're active — already near the floor for this aesthetic, 0 when idle), and (3) uncached TMDB proxy calls on detail pages.
